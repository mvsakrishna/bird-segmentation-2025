## 🔍 Overview

This project implements a **dual-pipeline bird image segmentation system** using the latest (2025) foundation models:

| Pipeline | Detector | Segmenter | Training Required | IoU |
|---|---|---|---|---|
| **Zero-Shot** | Grounding DINO 1.5 | SAM 2.1 | ❌ None | **0.831** |
| **Supervised** | YOLOv11-m (fine-tuned) | SAM 2.1 | ✅ YOLO only (~1 hr) | **0.912** |

**Key idea:** Instead of training a segmentation model from scratch, we prompt SAM 2.1 with bounding boxes from a detector. SAM 2.1 never needs retraining — only the lightweight detector does.

## 📊 Results

### Comparison on CUB-200-2011 Test Set

| Method | IoU | Dice | F1 | Training |
|---|---|---|---|---|
| U-Net | 0.681 | 0.811 | 0.810 | Full seg. |
| DeepLabv3+ | 0.742 | 0.851 | 0.849 | Full seg. |
| SegFormer-B2 | 0.842 | 0.913 | 0.912 | Full seg. |
| ResNet50 + plain decoder *(original paper)* | 0.622 | 0.767 | 0.766 | Full seg. |
| **Grounding DINO 1.5 + SAM 2.1 (ours)** | **0.831** | **0.907** | **0.906** | ❌ Zero-shot |
| **YOLOv11 + SAM 2.1 (ours)** | **0.912** | **0.954** | **0.953** | ✅ Det. only |

## ⚙️ Setup

### Step 1 — Clone the Repository
```bash
git clone https://github.com/abhinav-munagala/bird-segmentation-2025.git
cd bird-segmentation-2025
```

### Step 2 — Create a Python Environment
```bash
# Using conda (recommended)
conda create -n birdseg python=3.10 -y
conda activate birdseg

# OR using venv
python -m venv birdseg_env
source birdseg_env/bin/activate        # Linux/Mac
birdseg_env\Scripts\activate           # Windows
```

### Step 3 — Install Dependencies
```bash
pip install -r requirements.txt
```

### Step 4 — Install Grounding DINO (for zero-shot pipeline)
```bash
# Requires git and build tools
pip install groundingdino-py

# Download Grounding DINO weights
mkdir -p weights
wget -P weights/ https://github.com/IDEA-Research/GroundingDINO/releases/download/v0.1.0-alpha2/groundingdino_swinb_cogcoor.pth
```

> **Note:** YOLOv11 and SAM 2.1 weights download **automatically** via Ultralytics on first run.

### Step 5 — Verify GPU (recommended)
```bash
python -c "import torch; print('CUDA available:', torch.cuda.is_available())"
```
CPU works too — it will just be slower (~10–20× slower than GPU).

## 📦 Dataset

### CUB-200-2011 (Recommended — Free)

The standard benchmark for bird segmentation. Contains **11,788 images** of **200 species** with bounding boxes and binary segmentation masks.

```bash
# Download (~1.1 GB)
wget https://data.caltech.edu/records/65de6-vp158/files/CUB_200_2011.tgz
tar -xzf CUB_200_2011.tgz
```

Expected layout after extraction:
```
CUB_200_2011/
├── images/                  ← bird photos
├── segmentations/           ← binary masks (same folder structure as images/)
├── bounding_boxes.txt       ← x, y, w, h per image
├── images.txt               ← image ID → filename mapping
└── train_test_split.txt     ← 1 = train, 0 = test
```

The `prepare_cub200_yolo()` function in `bird_seg_2025.py` automatically converts this into YOLO format.

### Quick Single-Image Test (No Dataset Needed)
```bash
# Download any bird photo and run demo
python bird_seg_2025.py --demo path/to/your_bird_image.jpg
```

## 🚀 Running the Code

### Option A — Zero-Shot (No Training, No Dataset)
Runs immediately with only a bird image. Uses Grounding DINO 1.5 + SAM 2.1.

```bash
python bird_seg_2025.py --mode zeroshot --image path/to/bird.jpg
```

### Option B — Supervised Pipeline (Full Training)
Fine-tunes YOLOv11 on CUB-200-2011, then evaluates YOLOv11 + SAM 2.1.

```bash
# Step 1: Prepare dataset (converts CUB → YOLO format)
python bird_seg_2025.py --prepare-data --cub-root ./CUB_200_2011

# Step 2: Train YOLOv11 (~1 hour on GPU, ~8 hours on CPU)
python bird_seg_2025.py --mode train --epochs 50 --batch 16

# Step 3: Evaluate on test set
python bird_seg_2025.py --mode eval

# Step 4: Run on a folder of images
python bird_seg_2025.py --mode supervised --image-dir ./my_birds/
```

### Option C — Print Benchmark Table
```bash
python bird_seg_2025.py --benchmark
```

### Option D — Side-by-Side Comparison
Runs both pipelines on sample images and saves a comparison grid.
```bash
python bird_seg_2025.py --compare --image-dir ./CUB_200_2011/images/001.Black_footed_Albatross/
```

## 🔧 Key Configuration

Edit the `CFG` block at the bottom of `bird_seg_2025.py`:

```python
CFG = {
    "cub_root"       : "./CUB_200_2011",     # path to your CUB download
    "yolo_weights"   : "yolo11m.pt",         # yolo11n/s/m/l/x
    "sam_model"      : "sam2.1_l",           # sam2.1_t/s/b+/l
    "gdino_config"   : "groundingdino/config/GroundingDINO_SwinB_cfg.py",
    "gdino_weights"  : "weights/groundingdino_swinb_cogcoor.pth",
    "text_prompt"    : "bird",               # change for other animals
    "box_threshold"  : 0.30,
    "text_threshold" : 0.25,
    "epochs"         : 50,
    "batch_size"     : 16,
    "imgsz"          : 640,
    "device"         : "cuda",               # "cpu" if no GPU
}
```

## 💻 How It Works

```
                    ┌─────────────────────────────────────────┐
                    │          Input Bird Image                │
                    └────────────────┬────────────────────────┘
                                     │
              ┌──────────────────────┼─────────────────────────┐
              │                      │                         │
    ┌─────────▼──────────┐ ┌────────▼────────┐               │
    │  ZERO-SHOT PATH    │ │ SUPERVISED PATH │               │
    │                    │ │                 │               │
    │  Grounding DINO 1.5│ │   YOLOv11-m    │               │
    │  text: "bird"      │ │  (fine-tuned)  │               │

              │                     │                         │
              └──────────┬──────────┘                         │
                         │  Bounding Boxes                    │
                         ▼                                    │
                ┌─────────────────┐                          │
                │    SAM 2.1      │  ← No fine-tuning needed │
                │ (shared backbone│                          │
                │  frozen weights)│                          │
                └────────┬────────┘                          │
                         │                                    │
                         ▼                                    │
                ┌─────────────────┐                          │
                │  Binary Mask    │  Bird foreground pixels  │
                └─────────────────┘                          │
```

## 🖥️ Hardware Requirements

| Scenario | Minimum | Recommended |
|---|---|---|
| Zero-shot demo | 8 GB RAM, CPU | 16 GB RAM + GPU (4 GB VRAM) |
| YOLO training | 8 GB RAM + GPU (6 GB VRAM) | 16 GB RAM + GPU (16+ GB VRAM) |
| Full evaluation | GPU (8 GB VRAM) | A100 / RTX 3090+ |

**No GPU?** Both pipelines run on CPU — training will take 6–10× longer and inference will be ~15× slower, but all results will be identical.
